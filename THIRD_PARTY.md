# Third-Party Dependency Audit

The ACCT artifact uses small simulation and plotting dependencies listed in
`requirements.txt`, with optional EPyMARL smoke-test dependencies in
`requirements_epymarl_smoke.txt`. The file
`results/dependency_license_audit.csv` is generated by:

```bash
PYTHONPATH=. python3 experiments/audit_dependency_licenses.py
```

The audit records, for each requirement line, the installed distribution name,
version, and license metadata exposed by Python package metadata when the
package is installed. Missing entries are recorded as `MISSING_OR_OPTIONAL`
rather than treated as failures because some benchmark and EPyMARL dependencies
are optional for lightweight artifact checks.

This audit is a reviewer convenience, not a replacement for the authoritative
license files distributed by each dependency. Reviewers who need a full legal
license audit should inspect the package metadata and upstream repositories for
the exact versions installed in their environment.
